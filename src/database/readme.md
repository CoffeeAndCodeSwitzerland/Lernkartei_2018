============================================================================================================

	All Database Functions

============================================================================================================
	
	Inhalt
	------	
	- BackUp.java:		--> Funktionalität, um ein Backup aller Karten auf einen externen MySql Server
							zu machen. Funktion: backUp
	- Categories.java:	--> Funktionalität um Kategorien zu erstellen, löschen, abzufragen, usw.
	- Config.java:		--> Konfig Datenbank, welche Token und weitere Konfiginformationen enthält
	- Database.java:	--> Funktionalität, alle Karten abzuspeichern, zu löschen, bearbeiten und sie
							in Form einer ArrayList<String[]> abzurufen
	- Doors.java:		--> Enthält die Funktionalität um Türen zu erstellen und zu bearbeiten
	- Score.java:		--> Speichert, den Score, welcher von Yanis genutzt wird um später Infos über
							den Spieler zu erhalten, sowie Statistiken
	- Usercards.java:	--> Unused by Joel
	- Userlogin.java:	--> Unused by Joel
	
	Abhängigkeiten
	--------------
	- In diversen Models (DoorModel, CardModel, Usermodel) werden die Funktionen aus dem package 
	database verwendet. Die Funktionen doAction werden für Updates / Erstellen von Datensätzen 
	verwendet, und getData um die Daten einer bestimmten Tabelle zu holen.
	- Dieses package verwendet diverse Globals, Enviromentfunktionen und Debugger Funktionen
	grösstenteils die debug.Debugger.out() Funktion
	
============================================================================================================

	HowTo
	-----
	- Die meisten Funktionen werden nicht direkt verwendet, sondern über die Models
	- Um die Funktionen zu verwenden --> database.KLASSE.FUNKTION(PARAMETER);
